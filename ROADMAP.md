# hazumi 開発ロードマップ

「即タスク実行サポートアプリ」というコンセプトを、段階を追って形にしていくための計画。
個人開発前提で、各フェーズは「動くものを最短で出す → 触って学ぶ → 次に伸ばす」のサイクルで進める。

---

## Phase 0 — 土台を整える（now）

最初に "迷わず動く環境" を作る段階。コードを書く前に道具を揃える。

- [ ] README に書いた前提（Node バージョン、起動コマンド）が実際に動くことを確認
- [ ] `package.json` の `name` / `description` / `scripts` を整理
- [ ] `.env.example` を作って必要な環境変数の雛形を残す
- [ ] Vercel 側の自動デプロイが `main` push で走ることを確認
- [ ] 最低限の lint / format（ESLint + Prettier）を入れる

**完了条件**: `npm run dev` 一発で開発が始められて、push すると本番が更新される状態。

---

## Phase 1 — MVP（最低限「即タスク実行」を体験できる）

ユーザー（自分）が「思いついたことを即やり始められる」最小ループを動くようにする。

### 機能

- [ ] **タスク即時入力欄**: 開いた瞬間にフォーカスが当たっているテキスト入力
- [ ] **タスクの "細分化" 機能**: AI に渡して "今すぐ着手できる最小ステップ" を返してもらう（`api/` でサーバーレス API として実装）
- [ ] **着手ボタン**: 細分化された最初のステップだけを大きく表示し、それ以外を畳む
- [ ] **タイマー**: 5/15/25 分のシンプルな集中タイマー
- [ ] **完了アクション**: 終わったタスクを履歴に流す

### 非機能

- [ ] レスポンスが 2 秒以内に返ること（AI 応答含む）
- [ ] ページロードから入力可能になるまで 1 秒以内
- [ ] スマホブラウザで快適に使える（PWA 化は Phase 2）

**完了条件**: 自分自身が「思いついた → 30 秒以内に何かを始めている」を実現できている。

---

## Phase 2 — 個人開発として "続けたくなる" 形に

MVP を毎日触れる作りに整える。データの永続化と振り返りを足す。

- [ ] **ローカル保存 → クラウド保存**: 履歴をブラウザの IndexedDB に保存、後で Supabase などに同期
- [ ] **シンプルな認証**: Google OAuth など 1 タップで入れる方式
- [ ] **完了履歴ビュー**: 「今日／今週やったこと」の一覧
- [ ] **PWA 化**: ホーム画面に置いてネイティブアプリっぽく開ける
- [ ] **キーボードショートカット**: `Cmd+Enter` で着手、`Esc` でキャンセル など

**完了条件**: 1 週間以上、毎日自然に開いて使えている状態。

---

## Phase 3 — ネイティブ化を見据える

Web 版で形が定まったら、本来の "ネイティブアプリ" として作り直す or ラップする方針を決める。

### 検討ポイント

- **そのまま PWA で十分か** vs **ネイティブの利点が必要か**
  - ネイティブの利点: ホーム画面ウィジェット、ロック画面アクション、バックグラウンド通知、Siri / Shortcuts 連携、Apple Watch
- **採用候補**:
  - SwiftUI（iOS のみ、ネイティブ品質最大化）
  - Flutter / React Native（両 OS 対応、Web のロジック流用しやすい）
  - Capacitor（Vite で作った Web 版をそのままラップ）

### マイルストーン

- [ ] Web 版で最低 1 ヶ月使い込み、「これがあったら使い続ける」機能を 3 つ書き出す
- [ ] iOS 限定で SwiftUI プロトタイプを作る（または Capacitor で 1 日でラップしてみる）
- [ ] ホーム画面ウィジェットなど、ネイティブでしか出せない体験を 1 つ実装
- [ ] TestFlight で配布、自分以外の数人に試してもらう

**完了条件**: 自分が「Web 版より使いたい」と思えるネイティブ版がある。
